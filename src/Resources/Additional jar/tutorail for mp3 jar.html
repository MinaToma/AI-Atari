How To Use MP3Player Class

I want to share my success story today. During the last few days I had been trying create a simple mp3 player. I did some research and found Java Media Framework API (JMF), which I find it too complicated to setup. So I looked for other options then I found JLayer (jl1.0.jar) library, I had a little success with it and was able to play mp3 files, but that didn't stop me to keep looking for better alternatives. Then I ended up with JACo MP3 Player library (jaco-mp3-player-0.9.4.jar). 

Description:
JACo MP3 Player is a cross platform java mp3 player. 

Features:
 - very low CPU usage (~2%)
 - incredible small library (~90KB)
 - doesn't need JMF
 - easy to integrate in any application
 - easy to integrate in any web page (as applet).

For a complete list of its methods and attributes you can check its documentation here.

Looking at it, there's so much more features to add and it only keeps the fire burning. I found something to play with while learning new tricks.

Installing the library
If you're interested to try using the library, let me just give you something to start with. After downloading the jaco-mp3-player-0.9.4.jar, copy it to your ext folder of your java directory (C:\Program Files\Java\jdk1.6.0_03\jre\lib\ext).

Using the MP3Player class
Here's a sample code that I used as a basis for my first compile and run.

import jaco.mp3.player.MP3Player;
import java.io.File;

public class Example1 {
  public static void main(String[] args) {
    new MP3Player(new File("test.mp3")).play();
  }
}

Just make sure that your program and mp3 file are stored in the same directory for this code to work. Upon trying it, I encountered something unusual. The music played only within a duration of almost just a second then the program terminated.

The next thing I did was creating a very simple interface that consisted only of play button. Then I had to create a code that would execute the playing of mp3 file within a separate Thread, which is to prevent a problem that restricts the user from interacting with the interface while the music file is currently playing. So if we're going to modify the sample code, it would look like this:

// global declarations
MP3Player mp3_player;
PlayerThread player_thread;

// you will have to code this outside your main method
class PlayerThread extends Thread{
   public void run(){
      try{
         mp3_player = new MP3Player(new File("test.mp3"));
         mp3_player.play();
      }catch(Exception e){ System.err.println(e);}
   }   
}

// inside your actionPerformed() method of your listener define a thread then start
player_thread = new PlayerThread();
player_thread.start();

If it's not working, make sure you registered your listener to your play button.
